# sweep_cli


**sweep_cli** — это утилита командной строки (CLI) для поиска и перемещения файлов, написанная на Go. Проект создан для отработки навыков работы с файловой системой, параллельной обработкой данных и написанием тестов в Go.

## Возможности

*   **Поиск файлов**: Рекурсивный обход директорий для поиска файлов по заданным критериям (например, по расширению).
*   **Перемещение файлов**: Автоматическое или ручное перемещение найденных файлов в целевую директорию.
*   **Модульная архитектура**: Код разделен на логические пакеты (`search`, `cleanup`), что упрощает его тестирование и расширение.
*   **Юнит-тестирование**: Написаны тесты для ключевых компонентов (`main_test.go`), что гарантирует надежность базовой логики.

## Технологии

*   **Язык**: Go (Golang)
*   **Стандартные пакеты**: `os`, `filepath`, `io`, `testing`, `flag` (для парсинга аргументов командной строки)
*   **Инструменты**: Git, VS Code

## Установка и запуск

1.  **Клонируйте репозиторий**:
```bash
git clone https://github.com/CEs59C/sweep_cli.git
```
2.  **Перейдите в директорию проекта**:
```bash
cd sweep_cli
```
3.  **Соберите бинарный файл**:
```bash
go build -o sweep_cli .
```
4.  **Запустите утилиту** (пример):
```bash
./sweep_cli -source ./test_data -dest ./clean -ext .tmp
```

## Запуск тестов

Проект покрыт базовыми тестами. Для их запуска выполните:
```bash
go test -v
```
## Планы по развитию (Roadmap)

- [x] Базовая функциональность: рекурсивный поиск и перемещение файлов
- [x] Модульная структура и юнит-тестирование

### Ближайшие задачи (в работе)
- [ ] **Параллельная обработка**: Добавить горутины и каналы для конкурентного обхода директорий и обработки файлов. Использовать worker pool для контроля количества одновременно открытых файлов.
- [ ] **Контекст и graceful shutdown**: Внедрить `context.Context` для возможности отмены длительных операций (например, при нажатии Ctrl+C) и корректного завершения горутин.
- [ ] **Расширенный парсинг флагов**: Использовать пакет `flag` (или `cobra` в будущем) для гибкой настройки:
  - `-src` — исходная директория (можно указывать несколько)
  - `-dest` — целевая директория
  - `-ext` — расширения файлов для обработки (например, `.tmp,.log`)
  - `-mode` — режим работы: `move` (переместить), `copy` (копировать), `dry-run` (только показать, что будет сделано)
  - `-depth` — глубина рекурсивного поиска

### Перспективные идеи
- [ ] **Режим демона (фоновый сервис)**: Реализовать возможность запуска в качестве долгоживущего процесса, который по расписанию (например, через `time.Ticker` или крон-подобный планировщик) чистит заданные директории.
- [ ] **Поиск дубликатов**: Добавить функциональность поиска файлов-дубликатов:
  - Сравнение по размеру (быстый предварительный фильтр)
  - Вычисление хэша (SHA-256 или MD5) для точного сравнения
  - Опциональное перемещение дубликатов в отдельную папку или замена на hardlinks/symlinks
- [ ] **Логирование**: Добавить структурированное логирование (уровни: debug, info, error) с выводом в файл и/или stdout.
- [ ] **Конфигурационный файл**: Поддержка YAML/JSON конфига для сохранения часто используемых настроек.
